# TradingStrat Domain Glossary

This glossary defines the **Ubiquitous Language** used throughout the TradingStrat domain model. All terms should be used consistently across code, documentation, and conversations.

---

## Aggregate Roots

**Aggregate roots** are the entry points for accessing related entities. They enforce invariants and coordinate changes across the aggregate boundary.

- **Portfolio**: A container for investment positions and cash holdings. Enforces invariants like unique ticker positions and non-negative cash balance. Raises domain events when positions change.

---

## Entities

**Entities** have a unique identity that persists over time, even as their attributes change.

- **Position**: A holding of a specific security (ticker) within a portfolio. Tracks quantity, entry price, and entry date. Immutable ticker and portfolio association.

- **Trade**: An executed buy or sell transaction. Records ticker, type (Buy/Sell), quantity, price, commission, execution date, and notes.

- **HistoricalPrice**: A single OHLCV (Open, High, Low, Close, Volume) data point for a security at a specific date and timeframe. Used for backtesting and technical analysis.

- **Security**: A tradable financial instrument identified by ticker symbol and ISIN. Contains metadata like name, exchange, and currency.

- **CustomStrategy**: A user-defined trading strategy persisted to the database. Contains name, description, category, author, and a StrategyDefinition (rules, sizing mode).

- **PortfolioCashTransaction**: A record of cash deposit or withdrawal from a portfolio. Tracks type, amount, date, and notes.

- **BacktestResult**: The outcome of running a backtest simulation. Contains performance metrics, equity curve, trade list, and configuration.

- **LiveAnalysisResult**: Real-time analysis result for a security. Contains latest signal, current price, strategy-specific metrics (RSI levels, MACD values, etc.).

- **PerformanceMetrics**: Calculated performance statistics for a backtest or portfolio. Includes total return, annualized return, Sharpe ratio, max drawdown, win rate, profit factor.

- **EquityPoint**: A single point on an equity curve chart. Records date and portfolio value.

- **ChatMessage**: A message in the AI Trading Assistant conversation. Tracks role (User/Assistant/System), content, and timestamp.

- **StrategyRecommendation**: AI-generated strategy recommendation based on market analysis. Contains strategy type, confidence level, reasoning, and parameters.

## Value Objects

**Value objects** are immutable and defined by their attributes, not identity. Two value objects with the same attributes are considered equal.

- **TradeSignal**: An immutable trading decision (Buy/Sell/Hold) with price, quantity, and reasoning. Generated by strategies.

- **PortfolioSnapshot**: A point-in-time view of portfolio value and allocations. Contains total value, cash, position snapshots, unrealized gain/loss, and allocation percentages.

- **PositionSnapshot** (nested in PortfolioSnapshot): Snapshot of a single position with market value, allocation percentage, unrealized gain/loss.

- **RebalancingPlan**: Instructions for portfolio rebalancing to reach target allocations. Contains list of RebalancingSignals (Buy/Sell for each ticker), required cash, and executability status.

- **RebalancingSignal** (nested in RebalancingPlan): A single rebalancing instruction (ticker, action, quantity, price, total cost).

- **AllocationWeights**: Target allocation configuration for portfolio rebalancing. Dictionary of ticker → percentage + cash percentage. Must sum to 100%.

- **BacktestConfig**: Configuration for a backtest simulation. Defines ticker, date range, initial capital, commission rates.

- **PortfolioMetrics**: Portfolio-level performance metrics including total return, volatility, Sharpe ratio, and diversification ratio.

- **StrategyDefinition**: Complete definition of a custom rule-based strategy. Contains entry rules, exit rules, position sizing mode, and sizing parameters.

- **StrategyRule** (nested in StrategyDefinition): A single trading rule comparing an indicator to a value/indicator/price using a comparison operator. Rules can be combined with AND/OR logic.

- **TimeFrame**: Represents a time interval for market data (M1, M5, M15, M30, H1, H4, D1, W1, MN1). Immutable with factory methods for creation.

- **MarketFeatures**: Technical analysis features for ML prediction. Contains 26 indicator values as properties.

- **OptimizationResult**: Result of parameter optimization (Grid Search or Genetic Algorithm). Contains best parameters, best objective value, and all evaluated parameter sets.

- **StrategyComparison**: Comparative analysis of multiple strategies on the same data. Contains ranking by different metrics.

- **ComparisonRanking**: Ranking of a single strategy within a comparison. Contains strategy variant and score.

- **StrategyVariant**: A specific instance of a strategy type with particular parameters.

- **IchimokuSignals**: Signals generated by the Ichimoku Cloud strategy. Contains trend direction, entry mode, exit mode, and cloud state.

- **TrendState**: Classification of market trend (Bullish/Bearish/Neutral).

- **TradingStyle**: Classification of trading approach (Conservative/Moderate/Aggressive).

- **PredictionThresholds**: Thresholds for ML prediction interpretation (strong buy, buy, sell, strong sell).

- **ParameterSchema**: Schema defining valid parameter ranges for strategy optimization.

## Domain Events

**Domain events** represent significant occurrences in the domain. They are raised by aggregate roots and can trigger side effects in other parts of the system.

- **PortfolioCreatedEvent**: Raised when a new portfolio is created. Contains portfolio ID, name, and initial cash.

- **PositionAddedEvent**: Raised when a position is added to a portfolio. Contains portfolio ID, ticker, quantity, entry price, and entry date.

- **PositionRemovedEvent**: Raised when a position is removed from a portfolio. Contains portfolio ID, ticker, and quantity.

- **PositionQuantityChangedEvent**: Raised when a position quantity changes. Contains portfolio ID, ticker, old quantity, and new quantity.

- **CashTransactionRecordedEvent**: Raised when cash is deposited or withdrawn. Contains portfolio ID, transaction type, amount, and date.

- **PortfolioRebalancedEvent**: Raised when portfolio rebalancing is completed. Contains portfolio ID, signals executed, and timestamp.

- **TradeExecutedEvent**: Raised when a trade is executed during backtesting. Contains trade details (ID, ticker, type, quantity, price, commission).

- **StrategySignalGeneratedEvent**: Raised when a strategy generates a trading signal. Contains strategy name, ticker, signal type, and reasoning.

## Domain Services

**Domain services** contain domain logic that doesn't naturally belong to an entity or value object. They are stateless and operate on domain objects.

### Indicator Calculation

- **IIndicatorCalculator**: Domain service providing 26 technical indicators (SMA, EMA, RSI, MACD, ATR, Bollinger Bands, etc.). Single source of truth for all indicator calculations.

### Portfolio Management

- **PortfolioValuationService**: Calculates PortfolioSnapshot by applying current market prices to portfolio positions. Computes market values, allocations, and unrealized gains/losses.

- **PortfolioRebalancingService**: Calculates RebalancingPlan to transition from current allocations to target AllocationWeights. Generates buy/sell signals with commission costs.

- **PortfolioPerformanceService**: Calculates PortfolioMetrics from historical portfolio snapshots. Computes volatility, Sharpe ratio, and diversification metrics.

### Backtesting

- **PerformanceCalculator**: Calculates PerformanceMetrics from a backtest's equity curve and trade list. Computes returns, risk metrics, and trade statistics.

### Crossover Detection

- **CrossoverDetector**: Detects when price series or indicators cross thresholds or each other. Used by strategies for signal generation (MA crossover, MACD crossover, RSI crossing levels).

### Parameter Optimization

- **IParameterOptimizer**: Service for optimizing strategy parameters using Grid Search or Genetic Algorithms. Evaluates parameter combinations to maximize an objective (Sharpe ratio, total return, etc.).

### Multi-Timeframe Support

- **TimeframeAggregator**: Aggregates higher-timeframe (H1, H4, D1) data from intraday data (M1, M5, M15, M30). Constructs OHLCV bars for different timeframes.

### Machine Learning

- **IMLPredictionService**: Port interface for ML prediction. Abstracts ML.NET implementation from domain layer to maintain domain purity (zero external dependencies).

## Strategies

**Strategies** encapsulate trading logic. All strategies implement `IStrategy` and inherit from `BaseStrategy`.

- **BaseStrategy**: Abstract base class providing common functionality (position sizing, indicator access via IIndicatorCalculator).

- **RSIStrategy**: Trades based on Relative Strength Index overbought/oversold levels (default: oversold < 30, overbought > 70).

- **MACDStrategy**: Trades on MACD line crossing signal line (bullish crossover = buy, bearish crossover = sell).

- **MovingAverageCrossoverStrategy**: Trades when fast MA crosses slow MA (golden cross = buy, death cross = sell).

- **IchimokuStrategy**: Trades based on Ichimoku Cloud signals (conversion line, base line, leading/lagging spans, cloud position).

- **MachineLearningStrategy**: Uses ML.NET FastTree model to predict next-day returns based on 26 technical indicators. Walk-forward training to prevent look-ahead bias.

- **CustomRuleBasedStrategy**: Executes user-defined strategies by interpreting StrategyDefinition rules at runtime. Pre-calculates and caches indicators for performance.

## Specifications

**Specifications** encapsulate reusable business rules that can be tested independently.

- **ValidAllocationWeightsSpecification**: Validates that AllocationWeights sum to 100% (within 0.01% tolerance).

- **SufficientBacktestDataSpecification**: Validates that a date range contains enough data points for a strategy's minimum bar requirement.

---

## Domain Exceptions

**Domain exceptions** represent business rule violations.

- **DomainException**: Base class for all domain exceptions.

- **DuplicatePositionException**: Thrown when attempting to add a position with a ticker that already exists in the portfolio.

- **PositionNotFoundException**: Thrown when attempting to access a position that doesn't exist in the portfolio.

- **InsufficientCashException**: Thrown when attempting a transaction that would result in negative cash balance.

## Terminology Standards

### Preferred Terms (Use These)

- **Signal** (not "Decision" or "Recommendation") - for trading actions generated by strategies
- **Rebalancing** (not "Portfolio Adjustment" or "Reallocation") - for portfolio weight adjustments
- **TimeFrame** (not "Timeframe" or "TimeframeUnit" or "Interval") - for time intervals
- **Ticker** (not "Symbol" or "Stock Code") - for security identifiers
- **Position** (not "Holding" or "Stake") - for owned securities
- **Portfolio** (not "Account" or "Wallet") - for collection of positions
- **Entry/Exit Rules** (not "Buy/Sell Rules" or "Conditions") - for strategy logic
- **Backtest** (not "Simulation" or "Historical Test") - for strategy testing
- **Indicator** (not "Technical Indicator" unless distinguishing from fundamental) - for calculated metrics
- **Crossover** (not "Cross" or "Intersection") - for price/indicator crossing events
- **Oversold/Overbought** (not "Low/High RSI") - for RSI extremes
- **Snapshot** (not "View" or "State") - for point-in-time portfolio data
- **Aggregate Root** (not "Root Entity" or "Parent Entity") - DDD pattern terminology
- **Value Object** (not "VO" or "Immutable Object") - DDD pattern terminology
- **Domain Event** (not "Event" alone) - DDD pattern terminology
- **Use Case** (not "Service" or "Operation") - for application layer operations
- **Port** (not "Interface" when referring to hexagonal architecture) - for boundary abstractions
- **Adapter** (not "Implementation" when referring to hexagonal architecture) - for infrastructure implementations

### Deprecated Terms (Avoid These)

- ❌ "Decision" → Use "Signal"
- ❌ "Portfolio Adjustment" → Use "Rebalancing"
- ❌ "TimeframeUnit" → Use "TimeFrame"
- ❌ "Interval" → Use "TimeFrame"
- ❌ "Symbol" → Use "Ticker"
- ❌ "Holding" → Use "Position"
- ❌ "Account" → Use "Portfolio"
- ❌ "Simulation" → Use "Backtest"
- ❌ "Cross" → Use "Crossover"

## Naming Conventions

### Classes

- **Entities**: Singular nouns (Portfolio, Position, Trade)
- **Value Objects**: Descriptive nouns or noun phrases (TradeSignal, PortfolioSnapshot, AllocationWeights)
- **Domain Services**: [Purpose][Service/Calculator/Detector] (PortfolioValuationService, PerformanceCalculator, CrossoverDetector)
- **Strategies**: [StrategyName]Strategy (RSIStrategy, MACDStrategy)
- **Domain Events**: [AggregateRoot][Action]Event (PositionAddedEvent, PortfolioCreatedEvent)
- **Specifications**: [Purpose]Specification (ValidAllocationWeightsSpecification)
- **Exceptions**: [ErrorCondition]Exception (DuplicatePositionException, InsufficientCashException)

### Interfaces

- **Domain Services**: I[ServiceName] (IIndicatorCalculator, IMLPredictionService)
- **Strategies**: IStrategy (base interface for all strategies)
- **Specifications**: ISpecification<T>
- **Aggregate Roots**: IAggregateRoot
- **Ports**: I[Purpose]Port (IHistoricalDataPort, IMarketDataPort)

### Methods

- **Query Methods**: Get[Entity], Calculate[Metric], Is[Condition]
- **Command Methods**: Create[Entity], Add[Entity], Update[Entity], Remove[Entity], Record[Action]
- **Event Handlers**: On[Event] or Handle[Event]

### Properties

- **Entity IDs**: Id (not EntityId or EntityID)
- **Dates**: [Purpose]Date (EntryDate, TransactionDate, CreatedAt, LastUpdated)
- **Prices**: [Type]Price (EntryPrice, CurrentPrice, Close, High, Low, Open)
- **Collections**: Plural nouns (Positions, Trades, Rules)

## Architecture Principles

### Domain Layer Purity

The Domain layer MUST have **ZERO external dependencies** except .NET standard library. This ensures:
- Domain logic is portable and testable
- Business rules are not coupled to infrastructure concerns
- Domain can evolve independently of framework changes

**Example**: MachineLearningStrategy uses `IMLPredictionService` port instead of directly referencing ML.NET.

### Hexagonal Architecture (Ports & Adapters)

- **Ports** define interfaces for interacting with external systems (databases, APIs, ML models)
- **Adapters** implement these ports in the Infrastructure layer
- **Dependency flow**: Presentation → Application → Domain ← Infrastructure

### Ubiquitous Language in Code

All classes, methods, properties, and variables should use terms from this glossary. Code should read like domain expert conversations.

**Good Example**:
```csharp
RebalancingPlan plan = rebalancingService.CalculateRebalancing(
    snapshot,
    targetWeights,
    currentPrices,
    commissionPercentage
);
```

**Bad Example**:
```csharp
var adjustments = service.GetChanges(
    current,
    desired,
    prices,
    fees
);
```
